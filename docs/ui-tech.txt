Below is a **short, explicit UI-focused summary**, assuming full technical familiarity.

---

## UI Technologies & Frameworks

* **HTML**
  Standard HTML templates used for all pages (layout, tables, images, links).

* **CSS (vanilla)**
  External CSS files only, stored in Flask’s `static/` directory.
  Uses standard selectors (element, class, id), CSS variables, and basic layout styling.
  No CSS frameworks.

* **Flask (Python)**
  Backend framework responsible for routing and passing data to the UI layer.
  Does not render UI directly.

* **Jinja2**
  Templating engine for dynamic HTML generation (variables, loops, conditionals).

* **Static Asset Handling**
  CSS and images served via Flask `static/` directory and `url_for`.

---

## What Is **Not** Used (Crucial)

* No JavaScript or client-side logic
* No frontend frameworks (React, Vue, Angular)
* No CSS frameworks (Bootstrap, Tailwind, etc.)
* No build tools or bundlers
* No SPA or client-side state management
